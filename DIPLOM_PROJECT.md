## ВВЕДЕНИЕ

**Актуальность темы.** В современную эпоху цифровизации экономики и повсеместного внедрения облачных веб-технологий кардинальным образом меняется индустрия туризма и гостеприимства. С каждым годом возрастает туристический поток в Республику Адыгея — один из наиболее живописных и экологически чистых регионов Северного Кавказа. Миллионы туристов выбирают Адыгею ради посещения уникальных природных ландшафтов: плато Лаго-Наки, водопадов Руфабго, Хаджохской теснины, величественного горного массива Фишт, термальных источников посёлков Тульский и Цветочный, а также для погружения в этнографическую самобытность черкесской культуры и дегустации знаменитого адыгейского сыра.

Однако в отрасли информационного сопровождения туризма в регионе сохраняются выраженные проблемы разрозненности данных. Печатные буклеты мгновенно устаревают, государственные порталы перегружены сухим текстом и не поддерживают мобильную интерактивность, а глобальные сервисы планирования поездок лишены уникальных авторских эко-маршрутов с точным картографическим позиционированием и этнографическими рекомендациями. Таким образом, разработка интерактивного веб-путеводителя «Адыгея Гид» с встроенным ИИ-сопровождением является крайне актуальной задачей.

**Объект исследования:** Процесс разработки веб-приложений для интерактивного сопровождения туристической и экскурсионной деятельности в регионе.

**Предмет исследования:** Программные архитектурные решения, адаптивные СПА-интерфейсы на React, модели хранения СУБД NoSQL Cloud Firestore и механизмы сопряжения с генеративными большими языковыми моделями серии Gemini через прокси-сервер Express.

**Цель дипломной работы:** Разработать и экономически обосновать кроссплатформенную интерактивную систему «Адыгея Гид», объединяющую подробный каталог sights, редактор пользовательских отзывов, интерактивную карту высот, гастрономическую кухню и оперативного ИИ-экскурсовода в реальном времени.

**Задачи исследования:**
1. Провести детальное обследование предметной области и потребностей потенциальных туристов.
2. Провести бенчмаркинг аналогов на рынке информационных туристических систем.
3. Описать системные и архитектурные требования, разработать UML диаграммы Use Case и Deployment.
4. Разработать гибкую физическую структуру БД на базе Firestore, реализующую хранение оценок и отзывов.
5. Запрограммировать адаптивный фронтенд-модуль на React c использованием Tailwind CSS.
6. Разработать защищенный бэкенд на Node.js / Express для проксирования запросов на Google Gemini API.
7. Рассчитать инвестиционные и эксплуатационные расходы, налоги, Смч оборудования и время окупаемости для заказчика.
8. Подготовить тестовые пакеты и подтвердить работоспособность системы.

## 1 ТЕХНИЧЕСКИЙ ПРОЕКТ

### 1.1 Анализ предметной области
Туристско-рекреационный комплекс Республики Адыгея демонстрирует устойчивые темпы роста. Для успешного пребывания в регионе туристу требуется оперативно решать цепочку задач:
- Найти информацию об объекте с фотографиями и оценками других путешественников.
- Загрузить координаты для GPS навигатора.
- Выбрать маршрут, соответствующий уровню физической подготовки (от простых прогулок по тропам Руфабго до категорийных восхождений на Фишт).
- Изучить рестораны и кафе, где подаются национальные блюда.
- Контролировать прогноз погоды (особенно в горном кластере на Лаго-Наки).

Разрабатываемое веб-приложение призвано решить эти задачи в едином, интуитивно понятном bento-интерфейсе, доступном без скачивания из App Store/Play Store за счёт технологии адаптивного SPA.

### 1.2 Сравнительный анализ существующих решений
Для понимания рыночного позиционирования был проведен детальный анализ существующих решений по параметрам: адаптивность, интерактивность карт, этнографическое наполнение, обратная связь и элементы ИИ. Результаты сведены в таблицу.

*Таблица 1 – Результаты сравнительного анализа аналогов*

| Критерий оценки | Официальный ТИЦ Адыгеи | Яндекс.Путешествия | Разрабатываемый «Адыгея Гид» |
| :--- | :--- | :--- | :--- |
| **Адаптивность под смартфоны** | Низкая (верстка не оптимизирована) | Высокая | Высокая (резиновая сетка Tailwind) |
| **Наличие ИИ-ассистента** | Отсутствует | Общий чат-бот | Локальный ИИ-гид по Адыгее на Gemini |
| **Интерактивность карты** | Отсутствует | Высокая | Высокая (интеграция маркеров с карточками) |
| **Отзывы и оценки** | Отсутствуют | Высокая сложность фильтрации | Встроенная система рецензий |
| **Локальная кухня** | Только справочный текст | Отсутствует | Полная таблица блюд с рецептами |

### 1.3 Постановка задачи
Необходимо разработать интерактивный СПА-ресурс, реализующий:
1. Авторизацию и регистрацию пользователей.
2. Каталог природных достопримечательностей с фильтрацией по категориям: «Природа», «Культура», «Памятники», «Активный отдых», «Термы».
3. Раздел авторских треков и маршрутов с указанием времени перехода и уровня сложности.
4. Полноэкранную интерактивную карту с метками GPS и связью с детальным окном объектов.
5. Интерактивную панель кухни Адыгеи с рецептами (халюжи, адыгейский сыр, суп щипс) и адресами заведений.
6. ИИ-экскурсовода с контекстной памятью.
7. Интерактивный прогноз погоды по Майкопу и Лаго-Наки.
8. Инструмент администрирования для наполнения контента.

### 1.4 Моделирование предметной области
Проектирование велось с использованием методологии UML. Диаграмма вариантов использования (см. Приложение А) описывает роли актеров. Архитектурная диаграмма развертывания (см. Приложение Б) представляет трёхзвенную клиент-серверную топологию.

### 1.5 Выбор средств и технологии разработки
- **Язык разработки:** TypeScript (надежность, строгая типизация, снижение количества runtime-ошибок).
- **Фронтенд:** React 18 с Vite (высокая скорость сборки, виртуальный DOM, компонентный подход).
- **Стилизация:** Tailwind CSS (лучший утилитарный фреймворк для быстрой стилизации и адаптивной верстки).
- **Иконки:** Lucide-React.
- **Облачный бэкенд и СУБД:** Google Firebase Firestore, Firebase Authentication.
- **Интеграция ИИ:** Node.js Express прокси-сервер с Google GenAI SDK и моделью `gemini-2.5-flash`.

## 2 РАБОЧИЙ ПРОЕКТ

### 2.1 Физическая модель данных
База данных построена в NoSQL СУБД Cloud Firestore. Основные сущности описываются ниже:

*Таблица 2 – Схема коллекции «sights»*

| Поле в СУБД | Тип данных | Описание |
| :--- | :--- | :--- |
| **id** | string (UUID) | Уникальный идентификатор места |
| **title** | string | Наименование достопримечательности |
| **shortDescription** | string | Короткий анонс для карточки каталога |
| **description** | string (Markdown) | Большой текст с разметкой |
| **image** | string (URL/path) | Путь к изображению обложки |
| **category** | string | Категория (nature, culture, active, etc.) |
| **location** | string | Текстовый адрес |
| **coordinates** | map (lat: float, lng: float) | GPS маркер на карте |
| **rating** | number (float) | Усредненная оценка пользователей |

### 2.2 Функциональное взаимодействие модулей программы
При инициализации приложения клиентский код загружает данные из базы Firestore. При отправке сообщения в диалоговую форму ИИ-консультанта клиент осуществляет POST-запрос на `/api/ai-guide` передавая тело сообщения. Express-сервер читает токен, защищенный API-ключ `GEMINI_API_KEY` из `.env` файла, отправляет вызов в AI-модель и транслирует ответ обратно на клиент в формате JSON.

### 2.3 Входные и выходные данные
**Входные параметры:** Текстовые пользовательские запросы ИИ-гиду, поля формы регистрации, координаты геотрека, массив отзыва (text, rating).  
**Выходные параметры:** Отрендеренные в HTML карточки, интерактивная карта с плавными переходами к карточкам, текстовые диалоговые пузыри помощника на основе входящего JSON-стрима.

### 2.4 Установка и настройка программы
Для развертывания проекта в режиме разработки требуется:
1. Загрузить Node.js (версия v18 или выше).
2. Выполнить установку базовых npm пакетов: `npm install`.
3. Создать в корне проекта `.env` со следующим содержимым:
```env
GEMINI_API_KEY=AIzaSy...ваш_ключ_из_гугл_аи_студио
```
4. Выполнить сборку и запуск: `npm run dev`.

## 3 ЭКОНОМИЧЕСКОЕ ОБОСНОВАНИЕ

### 3.1 Исходные данные для расчета затрат
Разработка инвестиционного проекта «Адыгея Гид» выполнена силами двух человек за период в 2 месяца (60 дней).
- Исполнитель (Студент-разработчик): ФЗП = 65 000 руб/мес.
- Научный руководитель / Консультант: ФЗП = 85 000 руб/мес (время участия - 0.5 мес).

### 3.2 Расчет затрат на проектирование
Основная заработная плата с учетом регламентного премирования (20%):
$$ЗПосн = Поклад \times (1 + П) = (65000 \times 2 + 85000 \times 0.5) \times 1.2 = 172 \ 500 \times 1.2 = 207 \ 000 \ руб.$$

Дополнительная заработная плата (статья отпускных, 10%):
$$ЗПдоп = ЗПосн \times 0.10 = 207 \ 000 \times 0.10 = 20 \ 700 \ руб.$$

Общий фонд оплаты труда (ФОТ):
$$ФОТ = 207 \ 000 + 20 \ 700 = 227 \ 700 \ руб.$$

Отчисления на социальные нужды (30% во все внебюджетные фонды РФ):
$$Осн = ФОТ \times 0.30 = 227 \ 700 \times 0.30 = 68 \ 310 \ руб.$$

Расчет амортизации и машино-часа используемого ПК стоимостью 120 000 рублей:  
Действительный фонд времени оборудования равен 1820 часов. Годовая амортизация при норме 20% равна 24 000 рублей. С учетом потребления электроэнергии (0.45 кВт/ч при цене 6.5 руб/кВт) и накладных расходов (60%), суммарная стоимость работы ПК за период проектирования (200 часов) составила 18 340 рублей.

Себестоимость создания путеводителя:
$$Зпроект = ФОТ + Осн + tоб \times Смч = 227 \ 700 + 68 \ 310 + 18 \ 340 = 314 \ 350 \ руб.$$

### 3.3 Расчет затрат на внедрение
Затраты на внедрение составили 40% от ФОТ, включая кодирование и первичное тестирование в полевых условиях, а также транспортные эко-расходы на Лаго-Наки — 99 200 рублей.

### 3.4 Итоговый расчет прибыли и НДС
Суммарные затраты на создание и запуск равны 413 550 рублей. При установленной рентабельности проекта Исполнителя в 25% чистая прибыль составит 103 388 рублей.  
Отпускная цена по договору с Министерством Туризма:
$$Цена_р = Себестоимость + Прибыль = 413 \ 550 + 103 \ 388 = 516 \ 938 \ руб.$$

Налог на добавленную стоимость (НДС 20%):
$$НДС = 516 \ 938 \times 0.20 = 103 \ 388 \ руб.$$

Общая итоговая стоимость контракта с НДС:
$$Итого = 516 \ 938 + 103 \ 388 = 620 \ 326 \ руб.$$

Выгода от внедрения системы выражается в экономии на бумажных путеводителях, замене физических гидов и привлечении дополнительных 5 000 туристов в Адыгею ежегодно. Годовая выгода оценивается заказчиком в 380 000 рублей, откуда расчетный срок окупаемости инвестиционного проекта составляет:
$$Ток = 620 \ 326 / 380 \ 000 = 1.63 \ года \ (около \ 20 \ месяцев).$$
